# 🪐 EvoPlanet: Physics-Informed AI for Exoplanet Discovery

**Teaching AI the laws of physics to discover hidden exoplanets and eliminate false positives in massive space telescope datasets.**

EvoPlanet is an end-to-end scientific AI pipeline designed for the ISRO Hackathon. It detects extremely weak exoplanet transit dips hidden in noisy stellar light curves. Moving beyond standard "black box" deep learning classifiers, EvoPlanet employs **Physics-Informed Neural Networks**, **Multi-Modal Data Ingestion**, **Neural Architecture Search (NAS)**, and **Explainable AI (XAI)** to identify genuine exoplanet candidates and reject optical illusions like Background Eclipsing Binaries.

---

## 🚀 Key Innovations

### 1. EvoPhysics Inspector (Adaptive Physics Engine)
Standard AI often predicts "planets" that violate physical laws (e.g., predicting an impossibly large transit depth for a massive star). EvoPlanet introduces an `EvoPhysicsInspector` layer that cross-checks predictions against an 8-dimensional astrophysics context (Stellar Metadata + Box Least Squares derivations). If a prediction violates Kepler's laws or physical boundaries, the network mathematically overrides the decision, heavily penalizing the false positive.

### 2. Multi-Modal Vision (Beyond 1D Light Curves)
Traditional methods feed 1D light curves directly into standard CNNs, making them highly sensitive to stellar noise and instrument drift. EvoPlanet simultaneously analyzes:
- **Flux:** The brightness of the star.
- **Centroids (X/Y):** Spatial wobble to detect Background Eclipsing Binaries (BEBs).
- **Background Radiation:** To filter out local cosmic noise.

### 3. Dynamic Gated Fusion (Transformer)
The primary detector is a Transformer Encoder that processes the raw light curve sequence. It utilizes a **Gated Cross-Attention Fusion Layer** to dynamically query latent morphological representations generated by a self-supervised Autoencoder, learning exactly *when* to trust structural features versus raw sequential data.

### 4. EvoNet Neural Architecture Search (NAS)
We don't guess hyperparameters or fusion topologies. EvoPlanet utilizes a custom Evolutionary Algorithm (`EvoNet`) to automatically search the architectural space. EvoNet evaluates populations of different network topologies (e.g., Cross-Attention vs. Gated, number of heads, dropout rates) based on F1-Score fitness, evolving the ultimate "Champion" architecture and preventing evolutionary bias.

### 5. Iterative 3-Sigma Preprocessing
Standard smoothing algorithms (like Savitzky-Golay) often destroy the very transits we are trying to find. EvoPlanet uses an iterative 3-sigma masking step *before* polynomial flattening, ensuring the true planet signal is preserved while stellar variability is removed.

### 6. Explainability & Trust
- **MC-Dropout** is utilized at inference time to generate epistemic uncertainty bounds.
- **Attention Rollout** generates a 1D Saliency Heatmap, showing astronomers *exactly* which data points the AI focused on to make its decision.

## 💻 Running the Pipeline

### 1. Interactive Dashboard (Streamlit)
To visualize the pipeline, inspect the Multi-Modal inputs, and view the Saliency Maps and Physics Consistency scores, launch the interactive dashboard:
```bash
python -m streamlit run app/dashboard.py
```

### 2. End-to-End Pipeline Evaluation
To run a full end-to-end evaluation with the test dataset and calculate overall metrics:
```bash
python evaluate.py
```

### 3. Neural Architecture Search (EvoNet)
To simulate the EvoNet proxy-fitness environment and watch the genetic algorithm evolve the Transformer architecture:
```bash
python src/evorank/optimizer.py
```

### 4. Real Kepler Data Validation
To evaluate the pipeline against real, confirmed *Kepler* data:
```bash
python real_data_eval.py
```

## 🔬 Scientific Validation
EvoPlanet is not a "black box." By generating candidate probabilities alongside **Physics Consistency Scores**, **Uncertainty Metrics**, and **Attention Heatmaps**, it functions as a transparent Candidate Discovery & Vetting tool. It significantly reduces the human hours required to vet False Positives in massive datasets like TESS and Kepler, acting as the ultimate bridge between Deep Learning and hard Astrophysics.
